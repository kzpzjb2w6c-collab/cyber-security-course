I first installed UTM on my Mac and created a new virtual machine using the `debian-13.1.0-arm64-netinst.iso` image as my Mac runs on different CPU architecture that Windows users use. After the normal Debian installation, I logged in as my user `nexus` inside the VM.

At first, `sudo` did not work for my user `nexus` (as I was not in the sudoers group).  
To fix this, I logged in as root and added myself to the `sudo` group.

```bash
su -
usermod -aG sudo nexus
exit
```
After this change, I can run admin commands as nexus using sudo and my own password. In a real multi-user system this is a strong permission, so youâ€™d only give it to trusted users.
